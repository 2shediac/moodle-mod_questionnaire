Skip Logic & Conditional branching

The Skip logic (aka Conditional branching) feature will jump a respondent over a group/page of questions that isn't relevant to them.

Skip logic simple example

  1. Page 1
    • Question 1 Do you own a car? Yes/No.
      • Answer Yes -> go to page 2
      • Answer No -> go to page 5 (i.e. skip over irrelevant questions in pages 2 through 4)
  2. Page 2
    • Question 2: What is the color of your car?
    • Question 3: Did you buy it new or second-hand?
      • Answer new -> go to page 3
      • Answer second-hand -> go to page 4
  3. Page 3
    • Question 4: Why did you buy your car new?
  4. Page 4
    • Question 5: Why did you buy your car second-hand?
  5. Page 5
    • Question 6: Do you sometimes use public transport to go to work?
    • etc.

Procedure to be followed to obtain the above "skip logic" flow

  • Create Q1 as a Yes/No question.
    • Question Name: own car. Note that you must give your question a name, as this name will be used later on in the Questionnaire for the list of Parent questions. Please make this name as short as possible.
    • Response Required: Yes
    • Question Text: Do you own a car?
  • Insert a Page Break ---
  • Create Q2 as a Text Box question type
    • Question Name: optional
    • Response Required: Yes or No
    • Parent Question : own car->Yes
    • Question Text: What color is your car?
  • Create Q3 as a Radio buttons or Dropdown box question type
    • Question Name: car status
    • Response Required: Yes
    • Parent Question : own car->Yes
    • Question Text: Did you buy it new or second-hand?
    • Possible answers:
      • new
      • second-hand
  • Create Q4 as a Text Box question type
      • Question Name: optional
      • Response Required: Yes or No
      • Parent Question : car status->new
      • Question Text: Why did you buy your car new?
  • Insert a Page Break---
  • Create Q5 as a Text Box question type
      • Question Name: optional
      • Response Required: Yes or No
      • Parent Question : car status->second-hand
      • Question Text: Why did you buy your car second-hand?
  • Insert a Page Break---
  • Question 6 (see above). Now continue creating questions normally, until you want to use Skip Logic again.

Note 1

A respondent can walk a questionnaire which contains one (or more) skip logic pages forward and backward, as any "normal" multi-pages questionnaire. This means that any answer can be changed at any time (until the Submit button is pressed on the final page). Obviously if a respondent changes their response to a question containing a "Parent Question" setting, this will take the respondent along a new, different path through the questionnaire.

Note 2

Once a questionnaire has been created containing Parent questions, moving or deleting those questions must be done with great care, not to destroy the Skip Logic flow.

On the Questions Editing page, in the Action column, the "Move this question" feature will not allow the user to move a Child question before its Parent question or a Parent question after its first Child question. That is to say, the "Move question here" boxes will not be displayed in those forbidden positions.

If you click on the Delete icon of a Parent question , i.e. a question which has one or more Child questions depending on it, a warning message will display those Child quesions, asking you to confirm that you want to delete the Parent question and its Child questions.

Note 3

Only the following 3 Choice question types can be used as Parent questions: Yes/No, Dropdown boxes and Radio buttons. However, as seen in the example above, questions of any type can use a Parent Question.

Note 4

On the Questions editing page, the Parent question names will be displayed in red just above each of their Child questions.

Note 5

On the Questions editing page, once you have created one or more Child questions, a Check Page breaks button will be displayed next to the Add selected question type button. You can click that button at any time of creating questions, to check that the required Page breaks have been created before your Child questions. Missing required Page breaks will be added in the correct positions. If you do not want to bother with adding Page breaks as you create Child questions, you can just continue to create your questions and wait until you have created all your questions before clicking that Check Page breaks button to do the job for you.
